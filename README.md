# Court schedule Telegram bot

Бот скачивает Excel-файл по ссылке, парсит лист **График судов**, сохраняет события в SQLite и шлёт напоминания за:
- 1 день
- 12 часов
- 6 часов
- 2 часа

Также бот умеет:
- обновлять таблицу раз в день по расписанию;
- показывать события на сегодня, завтра, на конкретную дату;
- делать принудительную синхронизацию командой `/reload`.

## Как запустить

```bash
cp .env.example .env
# отредактируй .env

docker compose up -d --build
```

## Команды бота

- `/start`
- `/today`
- `/tomorrow`
- `/date 2026-05-25`
- `/next`
- `/reload`

## Как бот обрабатывает дату

Бот ищет в колонке **Дата судебного заседания** первую дату формата:
- `25.05.26 в 14:05`
- `16.03.2026 в 15:20`
- `до 04.02.26 предоставить документы`

Что не распознаётся, пропускается:
- `-`
- `Дата не указана`
- пустые ячейки

Если время не указано, подставляется `DEFAULT_EVENT_TIME`.

## Что стоит улучшить дальше

1. Добавить whitelist по статусам, если не все даты в столбце реально являются заседаниями.
2. Добавить админ-команду `/debug_raw` для показа строк, которые бот не смог распарсить.
3. Перейти с polling на webhook, если бот будет работать под доменом.
4. Добавить поддержку нескольких листов или нескольких таблиц.
